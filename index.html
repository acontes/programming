version 0.9 (November 2001)

ProActive is a Java library for Parallel, Distributed, and Concurrent computing. In the framework of an active object model (an actor like model but with full control over the granularity of parallelism), from a reduced set of rather simple primitives, a comprehensive and versatile library is proposed for parallel, distributed, and concurrent programming, including mobility of computations.

In the absence of any syntactical extension, ProActive programmers write standard code. The library is itself extensible by the programmers, making the system open for adaptations and optimizations.

ProActive is only made of standard Java classes, and requires no changes to the Java Virtual Machine, no preprocessing or compiler modification. It is based on the RMI Java standard library (for the sake of portability), but offers many added features:

  • Active objects, Asynchronous calls, Messages (Request and Reply)
  • Migration, Mobile Agents
  • Seamless sequential, multi-threaded, and distributed programming (no need to derive from a library class to implement remotely accessible objects)
  • Reuse: polymorphism between standard object and Active objects
  • Automatic future-based synchronization (wait-by-necessity)
  • Libraries for sophisticated synchronization, and collaborative applications
  • Interfaced with RMI registry, and Jini
  • Compatible with Swing and AWT

Moreover, the library features several optimizations improving performances. For instance, whenever two active objects are located within the same virtual machine, a direct communication is always achieved, without going through the network stack. Furthermore, this optimization is ensured even when the co-location occurs after a migration of one or both of the active objects.
